Introduction

The JavaLog is a set of Java classes that formats and writes events to one or more log devices.  The JavaLog frees the programmer from coding unflexible "print" statements in his code.  It offers an abstract interface to the various log devices and allows the user to configure each log device at run time.  Additionally, JavaLog allows the user to dynamically filter and format the logged events to suit his particular needs at run time.

For example, if the programmer logs events to the JavaLog, the user may, at run time, decided that he wants to see error events in a pop-up dialog box, warning events written to a file, and informational events on the standard output.  Additionally, for each type of event (errors, warnings, notices, traces, or user defined) he may specify a format in which he wishes each event to be logged.

The programmers job is simple:

import grace.log.Log;
...
public void foobar(String data) {
    Log.trace(data);
}
The resulting program will then produce the following:
03/07/99 17:02:48: 13: trace: system.main.Thread1: Test.foobar:54: "MyData"
Yes, it's just the simple!  Notice that in the preceding call to Log.trace(data),  JavaLog will printed out the give String data and automatically determined and logged the following:
  • Time
  • Event sequence number
  • Event type
  • Thread group(s) and thread name.
  • Classname, function, and line number whence the log event was generated.
