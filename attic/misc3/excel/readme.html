Amortization Demo
This example application demonstrates how use Formula One on a web server to create dynamic content. This demo requires a web server that supports servlets as well as page compilation.

How to Build
First choose a location on your web server to install all of the files in the demo directory except DisplayJPGServlet.java. All of the files should be located together in a single directory. Then open up DisplayJPGServlet.java in a text editor and find where the file "mortgage.vts" is referenced. Edit this reference so that it points to the correct location of "mortgage.vts" on your server. This will be a local reference instead of a URL since DisplayJPGServlet will be running on your web server. Now compile DisplayJPGServlet. You will need to include the javax.servlet.* packages in your classpath as well as F1J8Swing.jar. The command will look something like the following depending on the location of these items on your system.

    javac -cp .;\jsdk2.0\jsdk.jar;\f1java\F1J8Swing.jar DisplayJPGServlet.java

How to Deploy
The first step in deployment was done before building the DisplayJPGServlet servlet by copying the files to your web server. Now look in the web server directory with the demo and edit the file "loan.jhtml". This file has a reference to mortgatge.vts which will need to be updated as was done to DisplayJPGServlet.java. Again, this will be a local reference instead of a URL. The embedded java gets compiled and executed on the server rather than on the client system. Once this file location is updated, you are done with deployment.

How to Execute
To use the demo simply point your browser to the amortization.htm file on your web server.
